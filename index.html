01
Why Capacitor? (vs Cordova)
Choosing the right bridge framework

Both Capacitor and Apache Cordova wrap your HTML/JS game in a native WebView. However, Capacitor is the modern successor and strongly recommended for new projects.

FeatureCapacitorCordova
Maintained byIonic Team (active)Apache (slow updates)
Native project structureFull native projectsHidden, abstracted
Android Studio / Xcode integrationFull, directLimited
Plugin ecosystemModern, actively updatedLarge but aging
Single-file HTML apps✓ Perfect fit✓ Works
App Store approval compatibilityExcellentGood
✅
Verdict: Use Capacitor
This entire guide uses Capacitor. It produces real Android Studio and Xcode projects, giving you full control and the best compatibility with modern store requirements.

Prerequisites & System Setup
Install everything you need before starting
⚠️
Platform Requirement
iOS builds require a Mac with Xcode. Android builds work on Windows, Mac, or Linux. If you only have Windows, you can still build for Android today and handle iOS later on a Mac.
🤖 For Android
Node.js 18+
Java JDK 17+
Android Studio (latest)
Android SDK (API 33+)
npm or yarn
🍎 For iOS
macOS 13+ (Ventura or later)
Xcode 15+
CocoaPods
Apple Developer Account ($99/yr)
Node.js 18+

Step-by-step System Setup

1
Install Node.js
Download from nodejs.org — install the LTS version (18 or 20). Verify after install:
bash
node --version   # Should show v18 or higher
npm --version    # Should show 9 or higher
2
Install Android Studio (for Android builds)
Download from developer.android.com/studio. During installation, ensure these SDK components are selected:
textAndroid Studio SDK Manager → SDK Platforms
✓ Android 13 (API 33) or higher
✓ Android SDK Build-Tools 33+
✓ Android SDK Platform-Tools
✓ Android Emulator (for testing)
Then set the environment variable so Capacitor can find your SDK:
bashmacOS / Linux: add to ~/.zshrc or ~/.bashrc
export ANDROID_SDK_ROOT="$HOME/Library/Android/sdk"  # macOS
export ANDROID_SDK_ROOT="/home/user/Android/Sdk"      # Linux
export PATH="$PATH:$ANDROID_SDK_ROOT/tools:$ANDROID_SDK_ROOT/platform-tools"
powershellWindows — System Environment Variables
ANDROID_SDK_ROOT = C:\Users\YourName\AppData\Local\Android\Sdk
3
Install Java JDK 17 (required for Android)
Download from adoptium.net (Eclipse Temurin, free). After install:
bash
java --version   # Should show 17 or higher
4
macOS only — Install Xcode & CocoaPods
Install Xcode from the Mac App Store. Then install CocoaPods (dependency manager for iOS):
bash
sudo gem install cocoapods
pod --version    # Verify install

# Accept Xcode license (required once)
sudo xcodebuild -license accept
03
Project Setup & Wrapping Your Game
Create the Capacitor project and add your HTML game
1
Create Project Folder & Initialize npm
bash
# Create and enter project directory
mkdir dont-trust-instructions
cd dont-trust-instructions

# Initialize npm project
npm init -y
2
Install Capacitor
bash
npm install @capacitor/core
npm install -D @capacitor/cli

# Install platform packages
npm install @capacitor/android
npm install @capacitor/ios   # macOS only
3
Create Your Web Root Folder
Create a www/ folder and place your game's HTML file inside it, renamed to index.html:
bash
mkdir www

# Copy your game file in as index.html
# (rename dont-trust-instructions.html → index.html)
cp dont-trust-instructions.html www/index.html
Your folder structure should now look like:
textProject structure
dont-trust-instructions/
├── www/
│   └── index.html        ← Your game
├── node_modules/
└── package.json
4
Initialize Capacitor
bash
npx cap init

# It will ask you three questions:
#  App Name:    Don't Trust the Instructions
#  App ID:      com.yourdomain.donttrust   ← use your own domain
#  Web dir:     www
ℹ️
App ID Format
The App ID (e.g. com.yourname.donttrust) must be globally unique across all app stores. Use reverse domain notation. If you don't own a domain, use com.github.yourgithubname.donttrust.
5
Add Platforms
bash
# Add Android
npx cap add android

# Add iOS (macOS only)
npx cap add ios
This generates android/ and ios/ directories — full native projects you can open in Android Studio and Xcode.
6
Sync Your Game to Both Platforms
Every time you edit your HTML game file, run this to push the changes into the native projects:
bash
npx cap sync
04
App Configuration & Assets
Icons, splash screens, permissions, and capacitor.config.ts

capacitor.config.json

After npx cap init, a capacitor.config.json is created. Update it with these recommended settings for a mobile game:

jsoncapacitor.config.json
{
  "appId": "com.yourdomain.donttrust",
  "appName": "Don't Trust the Instructions",
  "webDir": "www",
  "server": {
    "androidScheme": "https"
  },
  "android": {
    "backgroundColor": "#060b14",
    "allowMixedContent": false
  },
  "ios": {
    "backgroundColor": "#060b14",
    "contentInset": "automatic"
  },
  "plugins": {
    "SplashScreen": {
      "launchShowDuration": 1500,
      "backgroundColor": "#060b14",
      "showSpinner": false
    }
  }
}

App Icons & Splash Screens

You need icons in many sizes. The easiest way is using the official Capacitor Assets tool:

bash
npm install -D @capacitor/assets

# Create the assets folder and drop in ONE high-res icon:
mkdir assets
# Place your icon.png (1024×1024px, PNG) in assets/
# Place splash.png (2732×2732px, PNG) in assets/

# Auto-generate all required sizes:
npx @capacitor/assets generate --iconBackgroundColor '#060b14' \
  --iconBackgroundColorDark '#060b14' \
  --splashBackgroundColor '#060b14'
🎨
Icon Design Tips
Your icon should be 1024×1024px with no transparency (App Store rejects transparent icons). Design a dark background (#060b14) with bold gold "DTTI" text or a skull emoji. Don't include rounded corners — the OS applies them.
05
Building for Android (AAB)
Generate the Android App Bundle for Play Store submission
🤖 Android Phase
1
Open the Android Project in Android Studio
bash
npx cap open android
# This launches Android Studio with your project
Android Studio will run a Gradle sync automatically. Wait for it to complete (may take a few minutes first time).
2
Set App Version
In Android Studio, open android/app/build.gradle and set:
groovyandroid/app/build.gradle
defaultConfig {
    applicationId "com.yourdomain.donttrust"
    minSdkVersion 22          // Android 5.1+
    targetSdkVersion 34
    versionCode 1             // Increment by 1 every release
    versionName "1.0.0"       // Human-readable version
}
3
Create a Signing Keystore
You must sign your app before uploading. Keep this keystore file safe forever — you cannot update your app without it.
bash
# Run this in your terminal (outside Android Studio)
keytool -genkey -v \
  -keystore donttrust-release.keystore \
  -alias donttrust \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000

# You'll be asked for:
#  - Keystore password (remember this!)
#  - Your name, organization, city, country
#  - Key password (can be same as keystore password)
🔐
Back Up Your Keystore Immediately
Store donttrust-release.keystore in cloud storage, a password manager, or multiple secure locations. If you lose it, you CANNOT push updates to your existing app. Google cannot recover it for you.
4
Configure Signing in build.gradle
Add your keystore details to android/app/build.gradle:
groovyandroid/app/build.gradle — add signingConfigs block
android {
    signingConfigs {
        release {
            keyAlias "donttrust"
            keyPassword "YOUR_KEY_PASSWORD"
            storeFile file("../../donttrust-release.keystore")
            storePassword "YOUR_STORE_PASSWORD"
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}
5
Build the Android App Bundle (AAB)
In Android Studio, go to Build → Generate Signed Bundle/APK → Android App Bundle → release. Or via command line:
bash
cd android
./gradlew bundleRelease

# Output file location:
# android/app/build/outputs/bundle/release/app-release.aab
The .aab file is what you upload to Google Play — NOT an APK.
6
Test on a Real Device Before Submitting
bash
# Connect your Android phone via USB, enable USB debugging
# Then run:
npx cap run android

# Or build APK for sideloading (testing only, not for Play Store):
./gradlew assembleRelease
# Output: android/app/build/outputs/apk/release/app-release.apk
06
Google Play Store Submission
Setting up your developer account and submitting your AAB
💳
One-Time Registration Fee
Google Play requires a one-time $25 USD developer registration fee at play.google.com/console. Paid once, valid forever for all future apps.
1
Create a Google Play Developer Account
Go to play.google.com/console, sign in with your Google account, pay the $25 fee, and complete the developer profile (name, address, contact).
2
Create a New App in Play Console
Click "Create app". Fill in: App name, Default language, App or Game (choose Game), Free or Paid.
3
Prepare Your Store Listing
Under Store presence → Main store listing, prepare these assets:
Short description (80 chars max)
e.g., "A deceptive reaction game. Trust nothing. Press everything."
Full description (4000 chars max)
Describe gameplay, features, difficulty scaling
Hi-res icon (512×512 PNG)
Feature Graphic (1024×500 PNG)
Banner shown at top of Play Store listing
Screenshots (min 2, phone screenshots required)
At least 1080×1920px (portrait) — take real screenshots from your phone
4
Complete All Required Policy Sections
Navigate through the left sidebar's checklist. Required sections include:
App content → Content rating
Complete the IARC questionnaire — your game is likely "Everyone" or "Everyone 10+"
App content → Target audience
Select age range (13+ is safe for this game)
App content → Data safety
Declare what data you collect. Since auth data is localStorage-only, select "No data collected from users" unless you add a backend
Pricing & distribution
Select countries where your app will be available
5
Upload Your AAB to Internal Testing First
Go to Testing → Internal testing → Create new release. Upload your app-release.aab file. Add testers (your own email is fine). This lets you test the full Play Store install experience before going public.
6
Promote to Production
Once testing looks good, go to Production → Create new release, upload the same AAB, write release notes, and click Review release → Start rollout to production.
⏱
Review Time
First-time submissions typically take 3–7 days for Google review. Updates to existing apps usually take a few hours to 2 days.
07
Building for iOS (IPA)
macOS + Xcode required
🍎 iOS Phase — Mac Required
1
Open in Xcode
bash
npx cap open ios
# Launches Xcode with your iOS project
Xcode will open. Wait for it to index and resolve Swift packages.
2
Set Bundle Identifier & Version
In Xcode, click the App target in the left panel → General tab:
Bundle Identifier: com.yourdomain.donttrust
Version: 1.0.0
Build: 1
Deployment Target: iOS 14.0+
3
Set Up Apple Developer Account Signing
You need a paid Apple Developer account ($99/yr at developer.apple.com). Then in Xcode:
Go to Xcode menu → Preferences → Accounts → Add your Apple ID → Select your Team. Then in the target's Signing & Capabilities tab, check Automatically manage signing and select your Team.
4
Configure App Capabilities
In Signing & Capabilities, your game doesn't need special capabilities (no Camera, no Push Notifications). Leave defaults. This makes App Store review faster.
5
Test on a Real iOS Device
Connect your iPhone, select it in the Xcode device dropdown, and hit Run (▶). This installs the app directly on your phone for testing.
6
Archive & Export for App Store
In Xcode: Product → Archive (make sure target is "Any iOS Device", not a simulator). Once archiving finishes, the Organizer window opens automatically.
Click Distribute App → App Store Connect → Upload. Follow the wizard — accept all defaults. Xcode will upload your IPA to App Store Connect.
08
Apple App Store Submission
App Store Connect setup and review process
💳
Annual Fee Required
Apple charges $99 USD/year for the Apple Developer Program at developer.apple.com/programs. This covers all iOS, macOS, tvOS, and watchOS apps.
1
Create App in App Store Connect
Go to appstoreconnect.apple.com → My Apps → + → New App. Fill in: Platform (iOS), Name, Primary Language, Bundle ID (must match Xcode), and SKU (any unique internal identifier, e.g. DTTI2024).
2
Prepare App Store Metadata
App name (30 chars max)
Subtitle (30 chars max) — e.g., "A Deceptive Reaction Game"
Description (4000 chars)
Keywords (100 chars, comma-separated) — reaction, casual, game, brain, color, reflex
Support URL (required — can be a simple GitHub page or Google Doc)
Privacy Policy URL (required even for free games)
Screenshots for iPhone (6.5" and 5.5" sizes required)
6.5" = 1284×2778px | 5.5" = 1242×2208px
3
Select Your Build
In App Store Connect, under Build, select the build you uploaded from Xcode. It may take 15–30 minutes to appear after upload.
4
Fill in App Privacy & Rating
Under App Privacy, declare what data your app collects. Since user accounts use localStorage only (no server), you can declare no data collected. For Content Rating, complete the questionnaire — your game is "4+" (no objectionable content).
5
Submit for Review
Click Add for Review, then Submit to App Review. Optionally add review notes for Apple (not visible to users).
⏱
Review Time
Apple typically reviews apps within 24–48 hours, though first submissions can take up to 7 days. You'll receive an email when approved or if there are issues.

Post-Launch & Pushing Updates
How to update your game after it's live

Every time you make changes to www/index.html, follow these steps to push the update:

bashStandard update workflow
# 1. Edit your game file (www/index.html)

# 2. Sync changes to native projects
npx cap sync

# 3. Open Android Studio and increment versionCode
#    versionCode 1 → 2 → 3 (must increase every Play Store upload)

# 4. Build new AAB
cd android && ./gradlew bundleRelease

# 5. Upload new AAB to Play Console → Production → New release

# 6. For iOS: open Xcode, increment Build number, Archive, Upload

Troubleshooting
Common errors and how to fix them
!
"Gradle build failed" or SDK not found
bash
# Verify ANDROID_SDK_ROOT is set correctly:
echo $ANDROID_SDK_ROOT

# In Android Studio → SDK Manager, accept all licenses:
yes | sdkmanager --licenses
!
White screen / blank app on device
Your HTML file has a reference to an external resource that can't load in the WebView. Check for:
bash
# In capacitor.config.json, ensure:
"server": { "androidScheme": "https" }

# Test via Chrome DevTools remote debugging:
# 1. Connect phone, open chrome://inspect in desktop Chrome
# 2. Click "inspect" under your app's webview
!
Google Fonts not loading on device
Your device might be offline or blocking the font CDN. Fix: download the fonts and bundle them inline in your HTML, or use font-display: swap with a system fallback. For production, embed fonts as base64 data URIs in the CSS to guarantee they load offline.
!
iOS: "No provisioning profiles found"
In Xcode, go to Signing & Capabilities → Team. Make sure your Apple Developer account is added in Xcode → Preferences → Accounts and the correct team is selected. Try Automatically manage signing.
!
App Store rejection: "Minimum functionality"
Apple can reject apps that seem "too simple." Strengthen your rejection appeal by ensuring your store listing clearly describes the gameplay depth, difficulty scaling, streak system, and the deception mechanic. Include a gameplay video in the review notes (not required but helps).
!
"cap sync" not picking up changes
bash
# Force a full sync:
npx cap sync --deployment

# Or copy manually:
npx cap copy android
npx cap copy ios
